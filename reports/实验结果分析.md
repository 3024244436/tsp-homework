> 对照论文：Bouman, Agatz, Schmidt. *Dynamic programming approaches for the traveling salesman problem with drone*. Networks, 2018.

## 一、实验数据说明

| 来源 | 实例数 | n 范围 | 分布类型 |
|------|--------|--------|----------|
| gen_instances.py 生成 | 9个 | 8, 10, 12 | uniform / 1-center / 2-center 各3个 |
| TSPLib 标准库 | 2个 | 14 (burma14), 16 (ulysses16) | 真实地理坐标 |
| FSU 标准测试集 | 1个 | 15 | 标准 TSP 测试集 |

**算法配置：**
- **DP**：精确动态规划（标准 DP，k = ∞），三遍 Pass 1/2/3，复杂度 O(3ⁿ·n²)
- **RDP**：限制型动态规划，L ∈ {1, 2, 3, 4}，限制每次 operation 中 truck-only 节点数 ≤ L
- **A\***：A\* 启发式搜索（论文 §3.4.2），在状态图上利用 MST 下界引导搜索，同样保证精确解
- 固定参数：truck_speed = 1.0，drone_speed = 2.0（α = 2），drone_endurance = 10⁹（无限续航）

**关键发现：DP vs A\* 交叉点**

## 五、三方算法对比总结

| 维度 | DP | RDP (k=1) | A\* |
|------|-----|-----------|------|
| 精确性 | ✅ 精确 | ⚠️ 近似（Δ%≤13.6%） | ✅ 精确 |
| 速度 (n≤14) | 最快 (1~651ms) | 更快 (0~151ms) | 最慢 (1~730ms) |
| 速度 (n=16) | 最慢 (11.3s) | **最快 (1.3s)** | 中等 (7.0s) |
| 适用场景 | n≤14，需精确解 | n≥14，接受少量误差 | n≥16，需精确解 |
| 论文对应 | DP k=∞ | DP + k 限制 | A\* k=∞ |

## 六、论文结论逐条验证

| # | 论文结论（§4） | 验证状态 | 本实验证据 |
|---|-------------|:------:|-----------|
| 1 | "DP 可求解比 IP 更大规模的实例" | ✅ | n=16 ulysses16 DP 11.3s、A\* 7.0s 精确求解（论文 IP 仅到 n=10） |

## 七、时间增长趋势

```
 n=8:  DP 1ms,     A* 1ms
n=10:  DP 8ms,     A* 13ms     ← A* M ST 下界开销开始显现
n=12:  DP 63ms,    A* 96ms
n=14:  DP 651ms,   A* 730ms    ← 差距快速收敛
n=15:  DP 2,378ms, A* 2,582ms  ← 几乎持平
n=16:  DP 11,293ms,A* 7,012ms  ← A* 反超！MST 剪枝在大 n 生效
```

DP 时间随 n 呈 O(3ⁿ·n²) 指数增长。A\* 在小 n 时因 MST 下界计算有额外开销，但在大 n 时搜索空间爆炸、MST 剪枝收益超过开销，**n=16 成为交叉点**。RDP 则通过限制 k 在 n=16 仅需 1.3s。
